Spacing

Options

Classy Bootstrap offers several variations to its padding and margins. To enable these, you must decide what to call your size options. Originally, we used the same types as Bootstrap, but this became confusing as the padding was "xs", but with bootstrap made it seem like it only applied when viewed with "xs" screen size. By default our sizes are 1-7. We avoid 0 becuase 0 is reserved for removing features in other extended class sets.

In your app.less file, after inserting @import "classy", you can overide the default size names with the less array: 

@size-list:   "1",    "2",    "3",    "4",    "5",    "6",    "7";

Then to set the various pixel size of the margins and padding, drop the following arrays in as well, with your chosen sizes

@pd-list:      3,          5,          7,          10,         15,         30,         60;
@mg-list:      3,          5,          7,          10,         15,         30,         60;

Padding

To use padding, follow the format of pd-{size-name} with the size-name that you decided in your size-list; By default the following are enabled.

.pd-1
.pd-2
.pd-3
.pd-4
.pd-5
.pd-6
.pd-7

Each padding size also has variations as well. Use a -v- to make the padding only vertical.

.pd-v-5

Use a -h- to make the padding only horizontal.

.pd-h-5

You can also apply padding one side at a time with the l,r,t,b, respectively left, right, top, bottom.

.pd-l-4
.pd-r-4
.pd-t-4
.pd-b-4

To remove padding from an element, you can replace the size name with n for none. This will override any of the other padding settings from Classy Bootstrap. These would look like pd-v-n or pd-t-n or simply pd-n to remove all padding.

Margins

Margins are the exact same as padding except pd becomes mg. So .pd-v-7 for the largest padding applied vertically would be .mg-v-7 to make that a margin instead. Be careful not to upset the bootstrap grids when using these margin setings.

Gutters

Because sometimes you want to remove gutters, you can apply the .no-gutters class to your row element. This will provide the following css: 

.row.no-gutters {
  margin-right: 0;
  margin-left: 0;
}
.row.no-gutters > [class^="col-"],
.row.no-gutters > [class*=" col-"] {
  padding-right: 0;
  padding-left: 0;
}

Similar effects can be achieved through mg-h-n and pd-h-n, but .no-gutters makes a common combo of classes more semantic and attuned to Bootstrap specifically.

Colors

Options

Every project has its own color scheme. In order to maximize efficiency, we can build a list of those default colors and then generate classes based on them, much like Bootstrap does with its helper classes (primary, success, warning, etc.). In your app.less file, underneath @import "classy", you can add colors in the following manner. The defaults are shown below.

@blue:    #00aef3;
@navy:    #0f2446;
@red:     #e31818;
@gray:    #f6f6f6;
@darkgray:  #CCCCCC;
@white:   #ffffff;
@black:   #000000;
@accent:   #262626;

This creates color variables that you can use anywhere you'd like in your less files.

To extend the classes, however, you also need to create two Less arrays, like so:

@cnames:  "blue",   "navy",   "red",    "gray",   "darkgray",   "white",  "black",  "accent";
@cvals:   "@{blue}",  "@{navy}",  "@{red}", "@{gray}",  "@{darkgray}",  "@{white}", "@{black}", "@{accent}";

The cnames array is the name array that will be inserted into the class. The cvals array simply calls the variable you made above and attaches them to the respective name
