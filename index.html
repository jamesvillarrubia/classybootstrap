Introduction

Why?

Many projects have tailored sections or elements that can be identified through HTML ID attributes or through wrapper classes. You might have a chunk of css that only applies to the immediate child divs inside #Jumbotron-Front or maybe you have a specific type of button that is called out with .red-branded-logo-button. Your CSS or LESS or SASS is then appropriately chunked out to match those specific elements.

At some point, however, on larger sites that persist over many years and have many different subsections and chunks, these "grouped" css divisions can be quite annoying. There's repetition, but you don't know where to even begin to abstract it. And every new addition probably had slightly different css applied - maybe 5px padding here but 7px padding on almost the same element on a different template chunk or section. You can imagine a scenario where every different type of headline banner on your site has its own template AND its own set of css. By breaking your components into template structures and dependent css structures, you now have to manage two related assets that are likely updated, processed, and deployed differently.

What's worse is when you update your template, you have to make sure you css is updated simultaneously - perhaps with some complex versioning tied to your deployment. Heaven forbid Cloudfront cache the assets at different times and now your beautiful banner looks ugly for the first 30min it's live.

Classy Bootstrap was developed because of that very issue. A large site was being developed with 10 unique content types and 50+ content developers and copy-editors across the organization. Some had significant HTML experience while others had none. Some needed to be able to edit HTML and others weren't to be trusted with that capability. In order to allow chunks of html to be added or edited through the admin interface (ExpressionEngine), we needed to either allow scoped/inline css that would likely confuse or break other parts of the site or we could try to remove most of the need for custom css all together. Thus Classy Bootstrap was born. It is a way to write well-branded chunks of html templates without having to update or recompile your core css. Plus, it was simple enough that basic HTML knowledge could become enough to keep your site's bits and pieces all "in brand" and keep your designers from calling you at 2am.

Of course, this level of CSS abstraction may be a bit insane for some projects, but we've found that once you start using Classy Bootstrap and you have to edit/deploy your css less and less, you start to see its appeal.

Check it out and let us know what you think on Twitter at @classybootstrap.

Installation

LESS

Download the classy.less file to your computer. Wherever you are pulling your bootstrap.less file, typically in an app.less, place @import "classy" beneath the @import "bootstrap".

For example, the less file I typically compile looks like this:

@import "font-awesome";
@import "bootstrap";
@import "bootstrap-select";
@import "classy";

/*
 *  Extra less classes and coding
 */

Though not recommended, you can also edit the bootstrap.less file. Make sure to place classy.less after the other imports.

/* ... rest of bootstrap.less ...
 * Components w/ JavaScript
 */
@import "modals.less";
@import "tooltip.less";
@import "popovers.less";
@import "carousel.less";

/* Utility classes */
@import "utilities.less";
@import "responsive-utilities.less";

@import "classy.less";

Spacing

Options

Classy Bootstrap offers several variations to its padding and margin classes. To enable any of these, however, you must first decide how to reference your size classifications. Originally, we used the same abbreviations as Bootstrap, but this became confusing - padding was meant to BE extra-small, not just only apply to "xs" screens.

To avoid confustion, we use numeric size names, much like h1 or h4. By default our sizes are 1-7. We avoid 0 because it is reserved for removing features in other extended class sets.

In your app.less file, after inserting @import "classy", you can overide the default size names with the less array: 

@size-list:   "1",    "2",    "3",    "4",    "5",    "6",    "7";

Then set the various pixel size of the margins and padding by dropping the following arrays in to your app.less with your chosen sizes:

@pd-list:      3,          5,          7,          10,         15,         30,         60;
@mg-list:      3,          5,          7,          10,         15,         30,         60;

IMPORTANT: Make sure that your size list and mg/pd lists have the exact same number of elements!

Padding

To use padding, follow the format of pd-{size-name} with the size-name that you decided in your size-list.

By default the following are enabled:

.pd-1
.pd-2
.pd-3
.pd-4
.pd-5
.pd-6
.pd-7

Each padding size also has variations as well. Use a -v- to make the padding only vertical.

.pd-v-5

Use a -h- to make the padding only horizontal.

.pd-h-5

You can also apply padding one side at a time with the l,r,t,b, respectively left, right, top, bottom.

.pd-l-4
.pd-r-4
.pd-t-4
.pd-b-4

To remove padding from an element, you can replace the size name with n for none. This will override any of the other padding settings from Classy Bootstrap. These would look like pd-v-n or pd-t-n or simply pd-n to remove all padding.

Margins

Margins are the exact same as padding except pd becomes mg. So .pd-v-7 for the largest padding applied vertically would be .mg-v-7 to make that a margin instead. Be careful not to upset the bootstrap grids when using these margin setings.

Gutters

To remove gutters, you can apply the .no-gutters class to your row element. This will provide the following css: 

.row.no-gutters {
  margin-right: 0;
  margin-left: 0;
}
.row.no-gutters > [class^="col-"],
.row.no-gutters > [class*=" col-"] {
  padding-right: 0;
  padding-left: 0;
}

Similar effects can be achieved through mg-h-n and pd-h-n, but .no-gutters makes a common combo of classes more semantic and attuned to Bootstrap specifically.

Colors

Options

Every project has its own color scheme. In order to make use of these color more efficient without tons of duplicate CSS or incorrect RGB values, we can build a list of those default colors and then generate classes based on them, much like Bootstrap does with its helper classes (primary, success, warning, etc.). In your app.less file, underneath @import "classy", you can add colors in the following manner. The defaults are shown below.

@blue: #00aef3;
@navy: #0f2446;
@red: #e31818;
@gray: #f6f6f6;
@darkgray: #CCCCCC;
@white: #ffffff;
@black: #000000;
@accent: #262626;

This creates color variables that you can use anywhere you'd like in your less files.

You might also consider prefixing these variables to avoid confusion and make the reference classy bootstrap specific.

@cb_blue: #00aef3;
@cb_navy: #0f2446;
@cb_red: #e31818;

To extend the classes, however, you also need to create two more LESS arrays, like so:

@cnames:  "blue",   "navy",   "red",    "gray",   "darkgray",   "white",  "black",  "accent";
@cvals:   "@{blue}",  "@{navy}",  "@{red}", "@{gray}",  "@{darkgray}",  "@{white}", "@{black}", "@{accent}";

The cnames array is the array of names will be inserted into the class names along with the appropriate prefixes. The cvals array simply calls the variable you made above and attaches them to the respective class name

Text

To set a text color for an element, add a class in the format tx-blue.

For example

.tx-blue
.tx-navy
.tx-red
.tx-darkgray
.tx-black

Backgrounds

Backgrounds are just like text, except with a bg prefix.

.bg-blue
.bg-navy
.bg-red
.bg-darkgray
.bg-black

Borders

Border colors are just like text and backgrounds, except with a bd prefix.

.bg-blue
.bg-navy
.bg-red
.bg-darkgray
.bg-black

Hovering

To alter the element's coloring on hover, you can add the -hv- to any of the text, background, or border color classes.

.bg-blue .tx-white .bd-red AND .bg-hv-red .tx-hv-black .bd-hv-black

Visited

To alter a link's visited color, just add -vs- to its respective class. For a black link that is blue when its been visited, add the classes tx-black tx-vs-blue.

Borders

Sizing

Border sizing works like padding or margins. There is a size-name attached to each border size. To alter the default border pixel widths, add a @bd-list array and change the values. The default values are shown below:

@bd-list:     1,      2,      3,      4,      5,      10,     15;

To give an element a border, simply use bd-{size-name}Here are some examples:

.bd-1
.bd-2
.bd-5

Removing

To remove a border, add the bd-n or bd-0 class.

Individual

To add a border to the left, right, bottom, top, vertical, or horizontal apply the same prefix codes as with padding and margins. For exampe:

.bd-l-3 .bd-blue
.bd-l-7 .bd-blue

Notice how the bd-blue automatically applies a 3px solid style.

To remove the default style, you need to zero out the generic border with a bd-n. Because the bd-n is placed after the coloring but before the left, right, top, bottom classes, you can override them to add a border on one side only.

.bd-l-3 .bd-n .bd-blue
.bd-l-7 .bd-n .bd-blue

Alignment

Basic

Bootstrap comes with some basic alignment capabilities. From the documentation they look like this:

Left aligned text.

Center aligned text.

Right aligned text.

Justified text.

No wrap text.

With this code:

<p class="text-left">Left aligned text.</p> <p class="text-center">Center aligned text.</p> <p class="text-right">Right aligned text.</p> <p class="text-justify">Justified text.</p> <p class="text-nowrap">No wrap text.</p>

Classy Bootstrap adds some additional functionality and tweaks the abbreviations for the standard bootstrap classes.

To implement Classy alignemnt, simply use tx-c or tx-center for center alignment. The other options are tx-ltx-rtx-j and the full names as well (tx-left).

By Screen Size

But Classy Bootstrap does more than shorten class names. You can also change the alignment by screen size.

Left on Medium and below. Right on Large screens.

<p class="tx-left-md tx-right-lg">Left on Medium and below. Right on Large screens.</p>

As the example shows, you just attach -xs, -sm, -md, -lg

You can also NEGATE an alignment with the following suffixes: -not-xs, -not-sm, -not-md, -not-lg. These classes will turn alignment back to "inherit" on those particular sizes but maintain the state alignment for all others

Right on all but medium.

Right inherited. Text center, but not on medium.

<p class="tx-right-not-md">Right on all but medium.</p> <div class="tx-right-xs"> <p class="tx-center-not-md">Right inherited. Text center, but not on medium.</p> </div>
